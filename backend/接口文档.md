# **互感器二次压降检测仪检定系统 \- 前后端接口文档 (v2.0 Final)**

## **1\. 文档说明 & 全局规约**

**项目名称**：互感器二次压降检测仪检定系统

**版本**：v2.0 (终极详细版)

**适用对象**：后端开发人员、前端开发人员

**基础路径 (Base URL)**：http://localhost:8080/api

### **1.1 通信协议**

* **传输协议**：HTTP/1.1  
* **数据格式**：application/json; charset=utf-8  
* **字符编码**：UTF-8  
* **时间格式**：  
  * 日期：yyyy-MM-dd (如 2023-10-01)  
  * 日期时间：yyyy-MM-dd HH:mm:ss (如 2023-10-01 12:00:00)

### **1.2 统一响应结构 (Result\<T\>)**

所有接口均返回以下 JSON 结构：

{  
  "code": 200,          // Integer, 业务状态码：200-成功, 500-系统异常, 401-未认证, 403-无权限, 400-参数错误  
  "message": "操作成功", // String, 提示信息  
  "data": {}            // T, 业务数据泛型，具体结构见各接口定义  
}

### **1.3 鉴权机制**

* **方式**：JWT (JSON Web Token)  
* **Header**：Authorization: Bearer \<token\_string\>  
* **Token Payload 包含信息**：userId, username, role

## **2\. 数据库设计补充 (系统日志表)**

对应报告需求，新增 sys\_log 表结构。

**表名**：sys\_log

| 字段名 | 类型 | 描述 |
| :---- | :---- | :---- |
| id | BIGINT | 主键ID |
| username | VARCHAR(50) | 操作人账号 |
| operation | VARCHAR(50) | 操作描述 (如: "新增设备") |
| method | VARCHAR(200) | 请求方法全路径 |
| params | TEXT | 请求参数 (JSON) |
| time | BIGINT | 执行耗时(ms) |
| ip | VARCHAR(64) | 客户端IP |
| create\_time | DATETIME | 操作时间 |

## **3\. 接口详细定义**

### **3.1 认证鉴权模块 (AuthController)**

#### **3.1.1 用户登录**

* **URL**: /auth/login  
* **Method**: POST  
* **描述**: 对应 sys\_user 表。  
* **请求参数 (Body)**:

| 参数名 | 类型 | 必填 | 描述 |
| :---- | :---- | :---- | :---- |
| username | String | 是 | 登录账号 |
| password | String | 是 | 密码 (建议传输前加密) |

* **响应示例**:

{  
  "code": 200,  
  "message": "登录成功",  
  "data": {  
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",  
    "userInfo": {  
      "id": 1,  
      "username": "admin",  
      "realName": "系统管理员", // 对应 sys\_user.real\_name  
      "role": 0,               // 0:管理员, 1:检测员  
      "status": 1              // 1:启用, 0:禁用  
    }  
  }  
}

#### **3.1.2 用户登出**

* **URL**: /auth/logout  
* **Method**: POST  
* **描述**: 清除服务端 Session 或 Token 状态。  
* **请求参数**: 无  
* **响应示例**: { "code": 200, "message": "退出成功", "data": null }

### **3.4 检定任务管理模块 (TaskController) \- 核心业务**

对应报告表 4-4 biz\_test\_task (主表) 和 表 4-5 biz\_test\_result (明细表)。

#### **3.4.1 提交检定任务 (归档)**

* **URL**: /task/submit  
* **Method**: POST  
* **描述**: 事务操作，同时写入任务主表和明细表。后端自动从 Token 获取 operatorId，无需前端传递。  
* **请求参数 (Body) \- TaskCreateDTO**:

| 参数名 | 类型 | 必填 | 描述 |
| :---- | :---- | :---- | :---- |
| deviceId | Long | 是 | 设备ID (关联 biz\_device.id) |
| meterPointId | String | 是 | 计量点编号 |
| deliverDate | Date | 是 | 送检日期 (yyyy-MM-dd) |
| testDate | Date | 是 | 测试日期 (yyyy-MM-dd HH:mm:ss) |
| temperature | BigDecimal | 是 | 环境温度 (保留1位小数) |
| humidity | BigDecimal | 是 | 环境湿度 (保留1位小数) |
| tanPhi | BigDecimal | 是 | tanφ (保留4位小数) |
| rPercent | BigDecimal | 是 | r% (保留3位小数) |
| resultList | Array | 是 | 明细数据列表 |

* **resultList 数组元素结构 (ResultItemDTO)**:

| 参数名 | 类型 | 必填 | 描述 |
| :---- | :---- | :---- | :---- |
| phase | String | 是 | 相别 (ao, bo, co) |
| standardId | Long | 是 | 对应的标准ID |
| valF | BigDecimal | 是 | 比差 f(%) |
| valDelta | BigDecimal | 是 | 角差 δ(分) |
| valDu | BigDecimal | 是 | 压降 dU(%) |
| valUpt | BigDecimal | 是 | Upt 电压 |
| valUyb | BigDecimal | 是 | Uyb 电压 |

* **请求体完整示例**:

{  
  "deviceId": 1001,  
  "meterPointId": "MP-2025-001",  
  "deliverDate": "2025-12-10",  
  "testDate": "2025-12-11 14:30:00",  
  "temperature": 23.5,  
  "humidity": 45.0,  
  "tanPhi": 0.0035,  
  "rPercent": 100.000,  
  "resultList": \[  
    {  
      "phase": "ao",  
      "standardId": 5,  
      "valF": 0.0020,  
      "valDelta": 1.500,  
      "valDu": 0.0500,  
      "valUpt": 57.700,  
      "valUyb": 57.700  
    },  
    {  
      "phase": "bo",  
      "standardId": 5,  
      "valF": 0.0015,  
      "valDelta": 1.200,  
      "valDu": 0.0400,  
      "valUpt": 57.700,  
      "valUyb": 57.700  
    },  
    {  
      "phase": "co",  
      "standardId": 5,  
      "valF": 0.0018,  
      "valDelta": 1.300,  
      "valDu": 0.0450,  
      "valUpt": 57.700,  
      "valUyb": 57.700  
    }  
  \]  
}
